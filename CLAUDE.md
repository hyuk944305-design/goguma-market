# 이 프로젝트 (고구마마켓)

개발 연습용 폴더입니다. 정식 서비스가 아니라 이것저것 만들어보며 배우는 공간이에요.

## 기술 스택 (사용하는 도구들)

- **Next.js** — 웹사이트(화면)를 만드는 도구
- **Supabase** — 회원가입/로그인, 데이터 저장(데이터베이스)을 담당하는 도구
- **GitHub** — 작성한 코드를 보관하는 저장소
- **Vercel** — 실제 인터넷 주소로 웹사이트를 띄워주는 배포 서비스

## 배포 방식 (어떻게 인터넷에 올라가나)

GitHub에 코드를 올리면(push 하면), 이미 연결된 Vercel이 **자동으로** 배포합니다.
- 즉, "GitHub에 올림 = 잠시 후 실제 사이트에 반영됨" 입니다.
- 별도로 배포 버튼을 누르거나 서버에 직접 올릴 필요가 없습니다.

# 작업할 때 지켜야 할 규칙

이 사용자는 비개발자입니다. 아래 규칙을 항상 지켜주세요.

1. **쉬운 말로 설명하기**
   개발 용어(예: 커밋, 마이그레이션, 환경변수 등)가 나오면 비개발자도 이해할 수 있게 풀어서 설명합니다.
   예) "마이그레이션 = 데이터베이스 구조(표 모양)를 바꾸는 작업"

2. **무엇을 하고 있는지 자주 알려주기**
   작업 중간중간 "지금 ○○를 하고 있어요", "다음은 △△를 할 거예요" 처럼
   현재 진행 상황을 자주 말해줍니다. 묵묵히 길게 작업만 하지 않습니다.

3. **지시한 것만, 단계적으로 수행**
   사용자가 시킨 일만 합니다. 시키지 않은 추가 작업을 임의로 하지 않습니다.
   여러 단계가 필요하면 한 번에 몰아서 하지 말고 단계별로 진행하며 확인합니다.

4. **할 수 있는 건 최대한 직접 처리**
   코드 작성, 파일 수정, 명령어 실행 등 내가 할 수 있는 일은 최대한 직접 끝냅니다.
   사람이 꼭 해야만 하는 일(예: 비밀번호 입력, 외부 사이트 버튼 클릭, 결제 승인 등)만
   골라서, 무엇을·왜·어떻게 해야 하는지 알기 쉽게 안내합니다.

5. **다 만들면 바로 확인할 수 있게 열어주기**
   개발(작업)이 끝나면 결과를 바로 눈으로 확인할 수 있도록,
   로컬 미리보기를 실행하거나 해당 화면/주소를 열어줍니다.
